Network Fundamentals

The most fundamental term and concepts needed to learn the modern day computer networks.
  • What is a computer network ?
    Computer Network & Types is a system of interconnected computers. Devices are linked to each other with the purpose of sharing resources.
  • Network Criteria
    Network Criteria refer to the set of objectives or requirements that a computer network must meet to be considered effective, efficient, and reliable. These criteria serve as guidelines for designing, implementing, and managing network infrastructures.
  • Network Topology
    Network Topology Topology means the way different devices are arranged to create a network for telecommunication.
  • Network Types
    Network Types categorize networks based on their geographic scope, ownership, and the technologies they employ. These classifications help in understanding the different characteristics, functionalities, and applications of various network architectures.
  • OSI Model
    OSI Model is a conceptual framework that standardizes the functions of a telecommunication or computing system into seven distinct layers.
  • TCP/IP Protocol Suite
    TCP/IP Protocol Suite is the model which is used in real applications in contrast to OSI model studies previously which is only a in theory model.

Physical Layer

The Physical Layer in OSI model holds the responsiblity of physically transfer the data in form of bits over the network.
  • Signals & It's Types
    Signals & It's Types are fundamental to understanding how data is transmitted and processed in various fields, including telecommunications, electronics, and computer networking.
  • Signal Impairment
    Signal Impairment refers to any degradation or alteration that a signal experiences as it travels from the transmitter to the receiver, impacting the quality and integrity of the transmitted information.
  • Digital Transmission
    Digital Transmission refers to the process of transmitting data from one place to another in the form of digital signals. Unlike analog transmission, where information is conveyed through continuous signals that vary in amplitude or frequency, digital transmission involves discrete signals (often binary, consisting of 0s and 1s) to represent information.
  • Analog Transmission
    Analog Transmission is a method of conveying information using continuous signals that vary in amplitude, frequency, or phase over time. It's an older form of communication but is still used in various applications today, particularly in audio and video transmission, as well as in certain types of telecommunications.
  • Multiplexing
    Multiplexing is a technique used in telecommunications and data communications to combine multiple signals into a single signal over a shared medium, such as a cable or a wireless channel. This allows multiple users or devices to share the communication channel efficiently, maximizing its capacity and utilization.
  • Transmission Media
    Transmission Media , also known as communication channels, are the physical pathways through which data is transmitted from one device to another in a telecommunications network. These media can be classified into two main categories: guided (or bounded) media and unguided (or unbounded) media.

Data Link Layer

  • Introduction to Datalink Layer
    Introduction to Datalink Layer discuss about the concept to nodes and links. There are different types of links and sublayer which datalink layer operates with.
  • Framing
    Framing is a process used to divide data bits into manageable pieces or frames before they are transmitted over the network.
  • Error Control
    Error Control in the context of data communication is a set of methods used to detect and correct errors that occur in the transmission of data across a network. These errors can be caused by various factors, including noise, interference, and signal attenuation. The Error correcting codes are some of the techniques to correct the errors encountered.
  • DLC Protocols
    DLC Protocols : Data Link Control (DLC) protocols are rules and conventions that manage the communication between devices across a data link layer in a network. These protocols ensure the reliable transmission of data between network nodes, providing mechanisms for addressing, error detection, error correction, and flow control.
  • Media Access Protocols
    Media Access Protocols are a sub-layer of the data link layer in the OSI model, responsible for controlling how devices in a network uniquely access and use the physical medium to transmit data.
  • Link-Layer Addressing
    Link-Layer Addressing is a fundamental aspect of networking that operates at the data link layer (Layer 2) of the OSI model. It is responsible for providing a unique identifier for each network interface within a local network segment.

Local Area Network

  • Ethernet
    Ethernet is a widely used networking technology that defines the standards for the physical and data link layers of the OSI model, enabling communication between devices in a local area network (LAN).
  • WIFI , IEEE 802.11 Project
    WIFI , IEEE 802.11 Project , commonly known as Wi-Fi, is a set of standards for wireless local area networking (WLAN) defined by the Institute of Electrical and Electronics Engineers (IEEE). The project encompasses a series of specifications that govern various aspects of wireless communication, including the physical layer (PHY) and the media access control (MAC) layer.
  • Bluetooth
    Bluetooth is a wireless technology standard designed for short-range communication between devices, enabling them to exchange data and communicate without the need for cables.

Wide Area Network

  • Telephone Networks
    Telephone Networks , also known as telecommunications networks, are systems designed for the transmission of voice, data, and multimedia over long distances using a combination of wired and wireless technologies.
  • Cable Networks
    Cable Networks refer to telecommunications networks that deliver television, internet, and telephone services over coaxial cables. These networks typically use a hybrid fiber-coaxial (HFC) infrastructure, where fiber optic cables connect to nodes in neighborhoods, and coaxial cables deliver services to individual subscribers.
  • Cellular Telephony
    Cellular Telephony , commonly known as mobile phone service or cellular communication, refers to the provision of voice and data services over a cellular network infrastructure. It enables users to make calls, send text messages, and access data services using mobile devices, such as smartphones, tablets, and feature phones.
  • Satellite Network
    Satellite Network is a telecommunications network that uses satellites orbiting the Earth to facilitate communication between distant locations. These networks are commonly used for various applications, including television broadcasting, internet access, telephone communication, and data transmission.

Connecting Device & Virtual LAN's

  • Connecting Devices
    Connecting Devices refers to the process of establishing communication links between electronic devices to enable data exchange, collaboration, and interaction. Various technologies and methods are employed to facilitate device connectivity, depending on factors such as proximity, compatibility, and desired functionality.
  • Virtual LAN's
    Virtual LAN's is a logical grouping of network devices within one or multiple physical LANs, irrespective of their physical location. VLANs enable network administrators to segment a network into multiple isolated broadcast domains, allowing them to control traffic flow and improve network security and performance.

Network Layer : Data Transfer

  • Services of Network Layer
    Services of Network Layer The Network Layer, the third layer of the OSI model, provides services that facilitate communication and data transmission between devices across different networks. Its primary function is to route data packets from the source device to the destination device, regardless of the underlying physical network topology.
  • Overview of Network Layer Protocols
    Overview of Network Layer Protocols provides a brief description about what we are going to study ahead.
  • Network Switching
    Network Switching is a method of transmitting data in which digital data is broken down into small packets before being transmitted. These packets are then sent individually across a network and reassembled at the destination.
  • Performance Criteria
    Performance Criteria includes the terms like delay , throughput and packet loss.
  • Internet Protocol Version 4
    Internet Protocol Version 4 is the fourth version of the Internet Protocol (IP), which is the principal communications protocol used for relaying datagrams (packets) across network boundaries. IPv4 is the most widely used protocol on the Internet and forms the foundation of most IP-based networks.
  • Internet Protocol Version 6
    Internet Protocol Version 6 is the most recent version of the Internet Protocol (IP), the core communication protocol of the Internet. It was developed to address the limitations of its predecessor, IPv4, and to accommodate the growing number of devices and addresses required by the expanding Internet.
  • Transition from IPV4 to IPV6
    Transition from IPV4 to IPV6 is an ongoing process driven by the exhaustion of IPv4 addresses and the need for a larger address space to accommodate the growing number of devices connected to the internet.

Network Layer : Routing of Packets

  • Idea and Goals
    Idea and Goals is to route the packets with minumum possible cost over a desired destination.
  • Routing Algorithms
    Routing Algorithms are crucial components of network infrastructure that determine the paths data packets take as they travel from a source to a destination across a network. These algorithms play a vital role in ensuring efficient, reliable, and timely delivery of data in computer networks. One the algorithm is Distance Vector Routing Algorithm.
  • Unicast Routing Protocols
    Unicast Routing Protocols are designed to determine the best path for forwarding unicast traffic from a single source to a single destination in a network. These protocols play a critical role in directing data packets across the network efficiently and reliably.
  • Multicast Routing
    Multicast Routing protocols are designed to efficiently deliver data from one source to multiple destinations across a network. Unlike unicast communication, where data is sent from one sender to one receiver, multicast communication allows a sender to transmit data to a group of recipients simultaneously.
  • Internet Group Management Protocol ( IGMP )
    Internet Group Management Protocol ( IGMP ) is a communication protocol used by IP hosts and adjacent multicast routers to manage multicast group memberships within a network. IGMP operates at the network layer (Layer 3) of the OSI model and is an integral part of multicast communication in IP networks, including the Internet.

Transport Layer

  • Transport Layer Services
    Transport Layer Services refer to the functionalities provided by the transport layer of the OSI (Open Systems Interconnection) model or the transport layer protocols in the TCP/IP model.
  • User Datagram Protocol
    User Datagram Protocol : UDP is a connectionless and lightweight transport layer protocol used in computer networking.
  • Transmission Control Protocol
    Transmission Control Protocol is a core protocol of the Internet Protocol Suite, responsible for providing reliable, connection-oriented communication between devices on a network.

Application Layer

  • Application Layer Services and Paradigms
    Application Layer Services and Paradigms
  • Client-Server Paradigm
    Client-Server Paradigm is a fundamental model for distributed computing, where computational tasks are divided between clients and servers in a networked environment.
  • World Wide Web & HTTP
    World Wide Web & HTTP : The World Wide Web (WWW) and HTTP (Hypertext Transfer Protocol) are foundational components of the modern internet, enabling the retrieval and delivery of web content.
  • FTP ( File Transfer Protocol )
    FTP ( File Transfer Protocol ) is a standard network protocol used for transferring files between a client and a server on a computer network. It is commonly used for uploading, downloading, and managing files on remote servers.
  • Electronic Mail
    Electronic Mail , commonly known as email, is a method of exchanging digital messages between individuals or groups using electronic devices and the internet.
  • TELNET
    TELNET is a network protocol used for interactive communication between two remote systems or devices over a computer network, typically the Internet.
  • SSH
    SSH (Secure Shell) is a cryptographic network protocol that allows secure remote access and control of networked devices and systems over an unsecured network.
  • Domain Name System ( DNS )
    DNS (Domain Name System) is a hierarchical distributed naming system for computers, services, or any resource connected to the Internet or a private network. It translates domain names, which are human-readable identifiers, into numerical IP addresses, which are used to locate and identify devices and services on the network.
  • Peer-to-Peer Paradigm
    Peer-to-Peer Paradigm is a decentralized model where individual nodes in the network, known as peers, interact directly with each other to share resources, services, or information without the need for centralized servers or intermediaries.
  • Socket Interface Programming
    Socket Interface Programming involves using a set of system calls and functions to create network communication between processes running on different hosts over a network.

How Multimedia Works Over Network ?

  • Compression
    Compression is a technique which is used to reduce the amount of data to be transferred over the network. The compression can be lossy as well as lossless compression.
  • Multimedia in Internet
    Multimedia in Internet includes the streaming of stored , live or interactive audio/videos data over the interconnected computers and devices.
  • Real-time Interactive Protocols
    Real-time Interactive Protocols include RTP (Real-time Transport Protocol), RTCP (Real-time Control Protocol), SIP (Session Initiation Protocol) protocols and H.323 also.

Network Management

  • Fundamentals of Network Management
    Fundamentals of Network Management introduces the fundamental terms and concepts used in Network Management.
  • Simple Network Management Protocol ( SNMP )
    Simple Network Management Protocol ( SNMP ) is a network management protocol used for monitoring and managing network devices, such as routers, switches, servers, printers, and other network-attached devices.
  • ASN.1 (Abstract Syntax Notation One)
    ASN.1 (Abstract Syntax Notation One) is a standard notation used to describe data structures and data encoding rules in a platform-independent and language-independent manner. It is widely used in telecommunications, computer networking, and cryptographic applications for defining data formats and protocols.

Cryptography & Network Security

Cryptography plays a crucial role in ensuring network security by providing mechanisms for confidentiality, integrity, authentication, and non-repudiation of data transmitted over networks.
  • Goals of Network Security
    Goals of Network Security are to protect the confidentiality, integrity, and availability of data and resources within a computer network.
  • CIA Triad
    CIA Triad is a foundational framework in the field of information security, representing three core principles or objectives that are essential for ensuring the security of information and data assets within an organization.
  • Message Authentication
    Message Authentication ensures that the message has not been altered during transmission and that it indeed originates from the expected sender.
  • Digital Signatures
    Digital Signatures are cryptographic techniques used to provide authenticity, integrity, and non-repudiation for digital messages or documents.
  • Entity Authentication
    Entity Authentication is a process used to verify the identity of a user or entity attempting to access a system, network, or application.
  • Key Management
    Key Management in network security involves the generation, distribution, storage, and disposal of cryptographic keys used to secure communication and data within a network.
  • Network Layer Security : IPSec
    Network Layer Security : IPSec also known as network security, focuses on protecting the communication and data transmitted between devices at the network layer (Layer 3) of the OSI model. It encompasses various techniques, protocols, and mechanisms to safeguard network infrastructure, prevent unauthorized access, and mitigate security threats.
  • Internet Key Exchange ( IKE )
    Internet Key Exchange ( IKE ) is a protocol used in IPsec (Internet Protocol Security) VPNs to establish and manage security associations (SAs) between two peers. It is responsible for negotiating cryptographic parameters and exchanging keys for secure communication.
  • Virtual Private Networks ( VPN )
    Virtual Private Networks ( VPN ) is a technology that enables secure and encrypted communication over a public network, such as the internet.
  • Transport Layer Security : SSL/TLS & Protocols
    Transport Layer Security : SSL/TLS & Protocols is a cryptographic protocol used to secure communication over a computer network. TLS encrypts data transmitted between two parties, ensuring confidentiality, integrity, and authenticity of the communication. It is the successor to the older Secure Sockets Layer (SSL) protocol.
  • Application Layer Security : PGP & S/MIME
    Application Layer Security : PGP & S/MIME - Application Layer Security protocols such as PGP (Pretty Good Privacy) and S/MIME (Secure/Multipurpose Internet Mail Extensions) are used to secure email communications, providing encryption, authentication, and integrity verification
  • Firewalls
    Firewalls are network security devices or software applications that monitor and control incoming and outgoing network traffic based on predetermined security rules. They act as a barrier between a trusted internal network and untrusted external networks (such as the internet), helping to prevent unauthorized access, data breaches, and network attacks.

    QoS Mechanisms are techniques used in computer networking to prioritize certain types of traffic over others, ensuring that critical applications receive the necessary resources and performance levels.
